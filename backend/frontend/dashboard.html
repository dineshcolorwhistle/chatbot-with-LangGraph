Node & Edge Details

1. State Machine Nodes

Node Name Type Responsibility / Behavior
welcome Static Entry Greets the visitor with an initial friendly message and introduces the chat helper.
intent_classifier Agent/Router Analyzes visitor message intent (off-topic, budget, contact, valid RAG query) to route flow.
off_topic Guardrail Triggered on irrelevant chatter. Politely redirects the conversation back to the professional context.
budget_response Guardrail Triggered on cost requests. Explains pricing is customized and shifts toward representative contact collection.
contact_response Guardrail Triggered on contact requests. Explains how the team works and proactively requests contact details.
rag_conversation Core Agent Queries Pinecone DB for relevant business answers, qualifications, and captures lead data models.
limit_warning Guardrail Warns the visitor of the message threshold limit (e.g. 10 messages max) and requests final details.
final_input Agent/Router Explicitly forces capturing missing fields (Name, Email, Company) before completion.
completed Terminal State Generates Requirement Blueprint, triggers Email dispatch via SMTP, and saves final lead structure to MongoDB.

2. State Transition Edges & Logic

From Node To Node Transition Edge Logic / Condition
[Session Entry] welcome / intent_classifier / final_input / completed Conditional Routing (route_by_stage): Routes new sessions to welcome, active conversations to intent_classifier, limit reached to final_input, and finished leads to completed.
welcome END Direct transition. Greets the visitor and returns control to the user client interface.
intent_classifier off_topic / budget_response / contact_response / rag_conversation Conditional Edge (classify_intent): If classified 'off_topic' → off_topic; if pricing → budget_response; if contact request → contact_response; if valid → rag_conversation.
off_topic END Direct transition. Renders off-topic redirection template and yields back to wait for user text.
budget_response END Direct transition. Renders pricing shield template and yields back to wait for user input.
contact_response END Direct transition. Renders request nudge text and yields back to wait for user details.
rag_conversation limit_warning / END Conditional Edge (check_message_limit): Checks user message counter. If user_message_count ≥ MAX_USER_MESSAGES, routes to limit_warning. Otherwise, routes to END (normal next step).
limit_warning END Direct transition. Prompts warning and sets stage to final_input for the next interaction.
final_input completed Direct transition. Captures remaining details, transitions lead stage state to completed.
completed END Direct transition. Finalizes requirements, sends confirmation mails and saves lead to DB.
